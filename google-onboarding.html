🔧 הגדרות חד-פעמיות למפתח (לפני שליחת הקישור למשתמשים)
כדי שכפתור "התחברות עם Google" הזה יעבוד בכלל, המפתח (מי שמארח את הכלי הזה) צריך להכין פעם אחת, במחשב:
  1. נכנסים ל-console.cloud.google.com, יוצרים פרויקט חדש.
  2. בתפריט "APIs & Services" → "Library" → מחפשים ומפעילים (Enable) את Apps Script API.
  3. "APIs & Services" → "OAuth consent screen" - בוחרים External, ממלאים שם אפליקציה + מייל, ומוסיפים את ה-scopes: script.projects ו-script.deployments. משאירים את האפליקציה במצב Testing ומוסיפים את כתובות המייל של המשתמשים כ-Test users (זה מה שגורם למסך "לא מאומת" - אפשר לעבור אימות רשמי בעתיד כדי להעלים אותו).
  4. "APIs & Services" → "Credentials" → "Create Credentials" → "OAuth client ID" → סוג Web application. תחת "Authorized JavaScript origins" מוסיפים את הכתובת שבה הקובץ הזה מתארח (למשל כתובת GitHub Pages שלכם).
  5. מעתיקים את ה-Client ID שנוצר, ומדביקים אותו למטה בקוד הזה במקום YOUR_CLIENT_ID_HERE.
זה שלב חד-פעמי; אחריו כל משתמש שתשלחו לו את הקישור פשוט לוחץ "התחברות עם Google" ומקבל כתובת מוכנה.
